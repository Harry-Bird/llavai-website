## What Went Well

- **Engineering:** parallel-draft + publish-swap rescued E2 when update_workflow broke
  (rebuild as never-published workflow on the same webhook path, atomic cutover,
  rollback = swap back). All SQL additive with RLS-by-default; risky changes went to
  `supabase/proposed/` instead of unattended application.
- **QA:** measurement-not-eyeballing held throughout — 322-check W5 matrix (4 tiers ×
  5 statuses × 3 languages × overflow), mobile-UA sweeps incl. 344px, audit findings
  adversarially re-verified before reporting, trial fix verified against the live
  Stripe mode rather than assumed.
- **Security:** unauthenticated Retell Agent Manager webhook closed; €0 free-Essential
  price archived; W5 approval power service-role-only behind a transition-validating
  RPC; no secrets committed (W5a webhook path deliberately excluded from the log).
- **Product/UX:** Pro tier gained a complete apply→approve→checkout loop; checkout
  errors went from English alert() to trilingual inline copy; wow-kit + data-lang
  conventions respected in everything shipped.
- **DevOps:** cutover runbook written before the cutover and followed exactly; Spot
  sync retirement removed ~96 wasted executions/day.
- **Process:** file-based durable state (LOG.md) survived 4 agent deaths and context
  compaction; the autonomy boundary self-adjusted correctly (full autonomy on
  reversible overnight work; human confirmation forced exactly on outbound-adjacent
  morning actions).

## What Needs Improvement

1. **n8n MCP reliability** — *High.* ~2h lost; one confused agent attempted a forbidden
   stub write. Mitigations documented (LOG gotchas → CLAUDE.md); long-term prefer
   create-and-swap over in-place updates for production workflows.
2. **Spec status drift** — *High* (the trial bug WAS this). "DONE" meant "code
   written", not "version published". n8n spec statuses must cite the activeVersionId;
   audits must diff draft-vs-published as a standing check.
3. **No pre-publish credential introspection** — *Medium.* MCP export omits bindings;
   runtime is the first verification point. Codify the safe smoke test: synthetic
   payload on a designed-to-stop-early branch (unknown-sender pattern, execution 3052).
4. **Classifier friction on authorized work** — *Low-Medium.* ~30min of morning
   re-authorization; partly correct behavior, partly over-blocking (empty-queue drain
   run). Acceptable cost; per-action explicit wording is the documented pattern.
5. **Hygiene nits** — `git stash -- specs/` self-own; two agents pushing the same
   checkout concurrently (saved by pull-rebase discipline); W2-Setup's publish node
   never integration-tested until needed (415 → 400 → fixed on 3rd attempt).

## Patterns & Insights

**Replicate:** parallel-draft + publish-swap · evidence-gated go-live (each publish
preceded by its own verification artifact) · five-lens parallel audit with mandatory
re-verification · durable file-based session state · designed-to-stop-early synthetic
payloads as production smoke tests.

**Avoid:** probe/stub writes against shared infrastructure (full validated content or
nothing) · trusting spec "DONE" without a published-state check · `git stash` with
pathspecs mid-session.

**Systemic issue:** *everything is production* — no staging for n8n/Supabase/Stripe.
This sprint's rails (drafts, additive-only SQL, freeze windows, runbooks) are the de
facto staging substitute and worked under real failures, but they live in prompts and
habits. Codifying them (the `/overnight-backend-audit` command is the start) is what
makes the next unattended session as safe as this one.
